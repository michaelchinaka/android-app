User Guide

Fridge Contents Page:

On this page is a list of items that are in your fridge. Each item is clickable and on click the view of a fridge item can expand. In the bottom right corner is a button for adding an item to your list of Frdige items. Clicking it takes you to a page where you can search for a food item and add that item to your list of fridge items. Below is a visual reprisentation of the Fridge contents page. Users can swipe left on an item to delete it from the list. Items in the fridge are stored in a database details are retained and the same if a user closes the app and returns

 

 

Once a fridge item has been clicked and expanded, its expanded details are shown. Purchase Date, Expiry Date, Quantity and Recurring purhcase attributes cxan be set here. Once values are set press the confirm button to save changes to local database. The date values are set by clicking on the date value in the button in line with the text. A date picker will pop up as seen in the image above and user can set the date value. Quantity value is an integer value that can be typed by user. Recurring purchase is a toggle that indicates whether a user would like that item to automatically be added to the shopping list once it expires.

 

Search Item Page:

At the top of the page is a search bar where a user can search a food item to add to the list of fridge items. After pressing enter results will load as seen in the image above and the user can simply click on the item they wish to add. A Toast will be presented to show the item has been added to the list of items. User can add more items if they wish or press the back button to return to view the fridge content items.

 

Expiring Soon Page:

On this page the user is presented with a lsit of food items expiring in the next 3 days. The user can click on thr checkbox or click anywhere on the row to select or unselectthe item. Once user is happy with items they have selected, they can press "Generate Recipe" button to take them to a new page that wills how the user suggestions of recipes they can make. 

 

Recipe Results Page:

Above as can be seen, the user is presented with recipe results they can make. Theres a name and image of the recipe, as well as number of missed, used and unused ingredients of the recipe based on the users selection of ingredients. The user can press the back button and edit their selection of ingredients, or click on a recipe and be presented with recipe information. 

 

Recipe Details Page:

Above is a read only description of the recipe the user has selected. It has a summary of the dish along with some ingredients to prepare the dish. In the menu bar theres a hamburger item the user can click on to take them to the clock app and set up a timer if they were cooking for example. Option can be seen below

 

Shopping List Page:

Above is an image of the shopping list page. Here users have a view of items on their shopping list. The user can tick the checkbox to mark the item has been bought. The user can add an item to the list by typing on the text input at the top of the page and clicking on the button alongsde it when they want to add to the list. Expired items that were set as recurrring in the fridge section of the app will automatically be added to this list. 

The user can swipe left to delete items from the list. Items on the list are stored in a database details are retained and the same if a user closes the app and returns

 

Settings Page:

On this page the user can set permissions for the app. They can allow access to the clock app for the function mentioned in the previous point.They can also turn on notifications that will tell them how many items are expiring soon when they open the app
